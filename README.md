# AI-ассистент для улучшения объявлений

---

### Запуск приложения
#### Первый способ
1. Клонируйте репозиторий `git clone <ссылка_на_проект>`
2. Создайте в корне проекта .env и заполните нужными переменными:
```sh
VITE_API_BASE_URL=`ссылка на бэкенд`
VITE_GIGACHAT_CREDENTIALS=`base64 ключ из личного кабинета GigaChat` 
``` 
Для получения API ключа для GigaChat (имейте в виду, что AI функционал работает только локально):
Перейдите в https://developers.sber.ru/studio/login, войдите в аккаунт, создайте новый проект и получите Authorization key в личном кабинете.
	
3. Установите зависимости `npm install`
4. Запустите проект `npm run dev`
5. Для финальной сборки проекта `npm run build`

Для запуска сервера:
1. Перейдите в папку server `cd server`
2. Скачайте зависимости `npm i`
3. Запустите командой `npm start`

#### Второй способ (через docker)
1. Выполните 1 и 2 пункты из первого способа
2. В терминале запустите команду `docker compose up --build` для старта приложения

### Самостоятельные решения
1. Бэкенд возвращал undefined id, исправил это
2. Убрал сортировку по цене, потому что не поддерживается на бэке
3. Функционал API запросов к GigaChat работает только локально в режиме разработки, потому что ключ не должен попасть в публичный доступ и такой функционал должен выполняться на бэкенде для защиты API-ключей и обхода CORS, а не фронтенде
4. Для плашки "Требует доработок" я не включал в расчет описание, так как в какой-то части документации было написано считать по параметрам, а также в макете, и лишь в одном месте было написано про описание.
5. API списка (/items) не возвращает характеристики и описание товара, поэтому на клиенте невозможно вычислить статус needsRevision для всего списка сразу. Поэтому на странице всех объявлений у меня needRevision всегда true, но я провожу детальную проверку на страницах просмотра и редактирования, где данных достаточно

### Возможные проблемы и недоработки
1. Реализован функционал темной темы для всех элементов на сайте, кроме notifications на странице редакторирования, они всегда с белым фоном. Также notifications не соответсвует дизайн-макету, но как я понял из задания, это не обязательное требование, поэтому сделал себе для них исключение
2. Можно улучшить функционал роутинга и вынести пути в отдельный конфиг, но для 3 путей я не сочел это нужным, если проект будет расширяться, в таком случае можно это исправить.
3. Я использовал SCSS для стилей, но использовал компоненты форм и кнопок из Ant Design. На самом деле для данного проекта можно было использовать только CSS, но мне захотелось использовать связку SCSS + Ant Design
